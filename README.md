## Project Objective

The objective of this project is to analyze ride-booking performance data to understand ride success, cancellations, and revenue patterns. The dashboard provides visibility into key operational metrics such as booking outcomes, vehicle performance, and payment behavior to support data-driven decision-making.

---

## Dataset Description

The project uses a ride-booking dataset containing over **100,000 records**, with key attributes including:

- Booking ID  
- Booking Date and Time  
- Booking Status  
- Vehicle Type  
- Payment Method  
- Ride Distance  
- Customer and Driver Ratings  

---

## Project Steps Followed

### Data Cleaning and Preparation (Excel & Power Query)
- Imported raw CSV data into Power BI.
- Handled missing values and standardized categorical fields.
- Converted date, time, and numeric columns to appropriate data types for accurate analysis.

### Data Modeling
- Reviewed the dataset structure to ensure fields were suitable for analysis and KPI calculation.
- Prepared the data for reporting by organizing columns used for filtering, aggregation, and time-based analysis.

### DAX Measures
- Created key measures including **Total Rides**, **Successful Rides**, **Success Rate (%)**, and **Total Revenue**.
- Used calculated measures to support KPI cards and visual analysis.

### Data Visualization
- Designed KPI cards for high-level performance tracking.
- Built charts to analyze ride outcomes, revenue distribution, and vehicle-type performance.
- Integrated slicers for **date range, vehicle type, and payment method** to enable interactive analysis.

---

## Key Insights

- **Operational Efficiency:** Identified overall ride success rate and highlighted the impact of cancellations on completed rides.
- **Revenue Distribution:** Observed balanced revenue contribution across different vehicle types, indicating stable demand.
- **Payment Trends:** Analyzed customer payment preferences to understand usage patterns.
